# 毕设启动文档

## 1. 题目建议

优先建议题目：

`算法竞赛训练管理系统的设计与实现`

可选题目：

- `面向 ACM 训练场景的管理系统设计与实现`
- `基于 Spring Boot 与 Vue3 的算法竞赛训练管理系统设计与实现`

说明：

- 如果学院更强调“系统设计与实现”，优先用第 1 个题目。
- 如果指导老师希望突出技术栈，可以用第 3 个题目。
- 模板对题目长度有约束，最终题目尽量控制在 25 字以内。

## 2. 选题定位

本项目不是普通的课程管理系统，而是面向算法竞赛训练场景的业务系统。系统围绕学生日常做题、比赛训练、教练管理三个核心问题展开，解决训练数据分散、训练效果难量化、比赛提醒不及时、学生状态难跟踪等问题。

系统采用前后端分离架构，前端负责训练信息展示、图表分析和交互操作，后端负责业务规则、数据同步、积分结算、推荐生成、异常预警和定时任务处理。项目适合作为“管理信息系统 + 数据分析规则”的软件工程类毕业设计。

## 3. 项目范围

### 3.1 用户角色

- 学生：查看个人画像、题目推荐、做题趋势、比赛历史、训练任务、积分记录和组队信息。
- 教练：管理学生账号、批量导入学生、同步 OJ 数据、查看异常提醒、管理队伍、发布比赛提醒和训练任务。

### 3.2 已实现的主要功能

- 用户登录与角色区分
- 训练任务管理
- 题单管理与完成标记
- 题库检索与做题状态展示
- 排行榜与积分流水
- 个人训练趋势与题型分布分析
- Codeforces / AtCoder 账号绑定与真实 OJ 数据同步
- AtCoder 提交 JSON 导入
- 战队创建、邀请与教练分配
- 官方比赛同步与训练赛提醒
- 训练异常预警与邮件通知
- 学生 Excel 批量导入与模板下载

### 3.3 技术栈

- 前端：Vue3、TypeScript、Pinia、Vue Router、Element Plus、ECharts
- 后端：Spring Boot 3、Spring Web、Spring Validation、Spring Data JPA、Spring Security、Spring Cache、Spring Mail
- 数据库：开发默认 H2，部署支持 MySQL
- 构建与部署：Maven、Vite、Docker、Docker Compose
- 外部数据源：Codeforces API、AtCoder 页面解析

## 5. 可直接使用的摘要草稿

### 5.1 中文摘要草稿

随着算法竞赛训练逐步走向团队化和数据化，传统依靠人工登记和口头反馈的训练管理方式已经难以满足日常训练组织、学生状态跟踪和比赛安排管理的需要。针对训练数据分散、学生做题情况难以量化、比赛提醒不及时以及教练管理成本较高等问题，设计并实现了一套算法竞赛训练管理系统。

本系统采用前后端分离架构，前端基于 Vue3、TypeScript 和 Element Plus 实现交互界面，后端基于 Spring Boot、Spring Data JPA 和 MySQL/H2 实现业务逻辑与数据持久化。系统面向学生和教练两类用户，提供训练任务管理、学生信息管理、队伍管理、排行榜、积分流水、题目推荐、比赛提醒、异常预警等功能。同时，系统对接 Codeforces 与 AtCoder 平台，实现用户公开数据同步，并支持 AtCoder 提交记录 JSON 导入，从而提升训练数据的真实性与完整性。在此基础上，系统结合评分、做题量等指标构建训练画像，生成个性化推荐题目，并通过定时任务和邮件服务实现比赛提醒与异常通知。

测试结果表明，该系统能够较好地满足算法竞赛训练场景下的日常管理需求，提升训练过程的信息化和可视化水平。该课题对竞赛训练团队的数字化管理具有一定的实际应用价值。

关键词：算法竞赛；训练管理；前后端分离；数据同步；题目推荐

### 5.2 英文摘要草稿

With the increasing trend toward team-based and data-driven algorithm contest training, traditional management methods based on manual records and verbal feedback can no longer meet the needs of training organization, student progress tracking, and contest scheduling. To address the problems of scattered training data, difficulty in evaluating problem-solving performance, delayed contest reminders, and high coaching management costs, an algorithm contest training management system is designed and implemented.

The system adopts a front-end and back-end separated architecture. The front end is built with Vue3, TypeScript, and Element Plus, while the back end is implemented with Spring Boot, Spring Data JPA, and MySQL/H2. The system serves both students and coaches, and provides functions including training task management, student information management, team management, ranking lists, point logs, problem recommendation, contest reminders, and abnormal training alerts. In addition, the system integrates with Codeforces and AtCoder to synchronize public user data, and supports importing AtCoder submission records in JSON format, which improves the authenticity and completeness of training data. Based on rating and problem-solving statistics, the system builds training profiles, generates personalized problem recommendations, and uses scheduled tasks and email services to send contest and alert notifications.

Test results show that the system can effectively support daily management in algorithm contest training scenarios and improve the informatization and visualization of the training process. The system has practical value for the digital management of competitive programming teams.

Keywords: algorithm contest; training management; front-end and back-end separation; data synchronization; problem recommendation

## 6. 技术路线与论文可写点

### 6.1 总体技术路线

1. 以前后端分离方式搭建系统基础框架。
2. 建立学生、任务、题目、比赛、队伍、积分、告警等核心数据模型。
3. 接入 Codeforces 与 AtCoder 数据源，形成真实训练数据同步链路。
4. 基于评分与做题量构建训练画像与隐藏分。
5. 围绕隐藏分实现题目推荐、题目分布分析与训练异常预警。
6. 通过定时任务和邮件通知实现比赛提醒与异常消息推送。

### 6.2 论文中值得重点写的“核心设计”

- OJ 数据同步机制：解决外部平台数据采集、解析、落库和更新问题。
- AtCoder JSON 导入机制：解决公开源做题明细不足的问题。
- 隐藏分模型：将 Codeforces、AtCoder 和做题量融合为训练水平指标。
- 推荐策略：围绕隐藏分生成热身、核心和挑战题目。
- 异常预警机制：根据训练异常规则生成提醒并支持邮件通知。
- 官方比赛同步机制：自动抓取 upcoming 比赛并按提醒规则推送。

### 6.3 可以写成“特色”但不要夸大的点

- 实现了面向真实训练场景的业务闭环，而不仅是单纯的数据录入。
- 系统具备外部平台数据接入、规则计算和提醒通知能力。
- 支持学生端和教练端双角色协同使用。

注意：

- 不要写成“采用智能推荐算法”或“实现高精度机器学习模型”，当前项目还达不到这个程度。
- 推荐与画像可以表述为“基于规则和训练画像的个性化推荐”。
